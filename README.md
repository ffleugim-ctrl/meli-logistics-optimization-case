# meli-logistics-optimization-case
Este projeto documenta uma intervenção real realizada no Centro de Distribuição do Mercado Livre (Cajamar/SP), onde apliquei análise de dados para solucionar um gargalo crítico de produtividade no setor de Devoluções (DV).
🚩 O Problema
Cenário: Backlog de 4.567 itens com risco de quebra de SLA (ETD 05).

Gargalo: Ineficiência no deslocamento dos colaboradores (tempo de "caça" sistêmica).

Equipe Inicial: 4 colaboradores com baixa densidade de tarefas.

💡 A Solução 
Implementação de Roteirização Física via Folhas de Tarefas.

Divisão da equipe para 8 colaboradores.

Criação de lotes fixos de 5 tarefas por folha, eliminando a dependência de busca aleatória no sistema.

📊 O Dashboard (Tech Stack: Looker Studio & Google Sheets)
O dashboard foi estruturado para monitorar três pilares:

Burn-down de Volume: Acompanhamento em tempo real da zeragem do backlog.

Performance Comparativa: Visão clara do ganho de produtividade entre o método padrão e a força-tarefa roteirizada.

People Analytics: Análise de produtividade individual (Peças/Hora) e alocação de headcount.

🚀 Resultados Alcançados
SLA: 100% cumprido dentro da janela operacional.

Produtividade: Aumento de 137% na eficiência individual (de 400 para 950 peças/hora).

Impacto: Eliminação total do backlog crítico em um único turno.
